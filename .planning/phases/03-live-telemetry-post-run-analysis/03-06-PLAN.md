    **3. Stamp GPS speed on FrameRecords at flush time:**
    In startPeriodicFlush(runID:), the current loop drains frames and calls flush(frames:). Extend this to stamp gpsSpeed on each frame before flushing:
    ```swift
    // Capture lastGPSSpeed on @MainActor before the detached task
    // (detached task cannot safely read @MainActor properties).
    // Pattern: capture the speed value as a local constant.
    ```

    PROBLEM: startPeriodicFlush uses Task.detached (off MainActor), but lastGPSSpeed is @MainActor. The current flush loop cannot read lastGPSSpeed directly.

    SOLUTION: Change startPeriodicFlush to a @MainActor non-detached periodic Task that reads lastGPSSpeed on each iteration, captures it, then passes the GPS value to a helper that stamps it into the drained frames before calling service.flush():

    ```swift
    private func startPeriodicFlush(runID: UUID) {
        guard let service = persistenceService else { return }
        let rb = ringBuffer
        periodicFlushTask = Task { @MainActor [weak self] in
            while !Task.isCancelled {
                guard let self else { return }
                if await rb.count >= 200 {
                    let frames = await rb.drain()
                    let gpsSpeed = self.lastGPSSpeed >= 0 ? self.lastGPSSpeed : nil
                    // Flush is performed on the service's ModelActor executor.
                    // Pass gpsSpeed as a stamp value for all frames in this batch.
                    Task.detached {
                        try? await service.flushWithGPS(frames: frames, gpsSpeed: gpsSpeed)
                    }
                }
                try? await Task.sleep(for: .seconds(2))
            }
        }
    }
    ```

    This requires adding a `flushWithGPS(frames:gpsSpeed:)` method to PersistenceService that stamps gpsSpeed on each FrameRecord before insert. Add to PersistenceService.swift:

    ```swift
    // SESS-02 + Phase 3: Batch insert with GPS speed stamped on each frame.
    func flushWithGPS(frames: [FilteredFrame], gpsSpeed: Double?) throws {
        modelContext.autosaveEnabled = false
        for frame in frames {
            let record = FrameRecord(from: frame)
            record.gpsSpeed = gpsSpeed
            modelContext.insert(record)
        }
        try modelContext.save()
    }
    ```

    Update emergencyFlush to also use flushWithGPS (passing nil for gpsSpeed, since at emergency flush time we may not have a fresh GPS reading):
    ```swift
    func emergencyFlush(ringBuffer: RingBuffer) async throws {
        let frames = await ringBuffer.drain()
        guard !frames.isEmpty else { return }
        try flushWithGPS(frames: frames, gpsSpeed: nil)
    }
    ```

    ALSO update the old `flush(frames:)` method or replace its callers with flushWithGPS. Keep `flush(frames:)` for backward compatibility with existing tests — just have it call `flushWithGPS(frames:gpsSpeed:nil)`.

    Create ArcticEdge/Today/ directory. Write TodayTabView.swift:

    ```swift
    // TodayTabView.swift
    // ArcticEdge
    //
    // Today tab: ContentView (session control) as the base.
    // Two overlays triggered automatically by AppModel state:
    //   - fullScreenCover: LiveTelemetryView when classifierStateLabel == "SKIING"
    //   - .sheet: PostRunAnalysisView when lastFinalizedRunID becomes non-nil
    //
    // The user never taps to open either overlay -- both are automatic.
    // Live view dismisses when classifierStateLabel transitions away from "SKIING".
    // Post-run sheet dismisses when user taps the X button (or swipes down).

    import SwiftUI

    struct TodayTabView: View {
        @Environment(AppModel.self) private var appModel
        @State private var showLive: Bool = false

        var body: some View {
            ContentView()
                .fullScreenCover(isPresented: $showLive) {
                    LiveTelemetryView()
                        .environment(appModel)
                }
                .sheet(item: Binding(
                    get: { appModel.lastFinalizedRunID },
                    set: { _ in }   // Dismiss is handled by PostRunAnalysisView's own X button
                )) { runID in
                    PostRunAnalysisView(runID: runID)
                        .environment(appModel)
                }
                .onChange(of: appModel.classifierStateLabel) { _, newLabel in
                    let isSkiing = (newLabel == "SKIING")
                    if isSkiing != showLive {
                        showLive = isSkiing
                    }
                }
        }
    }
    ```

    SHEET DISMISS: The `.sheet(item:)` modifier dismisses when the binding returns nil. However, `appModel.lastFinalizedRunID` is set once and stays set until endDay() clears it. This means the sheet would re-present every time it's dismissed mid-session. The correct behavior: once the user dismisses the post-run sheet, do NOT re-present it for the same run. Use a local @State var to track "already seen" run IDs:

    ```swift
    @State private var presentedRunID: UUID? = nil
    @State private var dismissedRunIDs: Set<UUID> = []

    // In body: observe lastFinalizedRunID changes
    .onChange(of: appModel.lastFinalizedRunID) { _, newID in
        guard let id = newID, !dismissedRunIDs.contains(id) else { return }
        presentedRunID = id
    }
    .sheet(item: $presentedRunID) { runID in
        PostRunAnalysisView(runID: runID)
            .environment(appModel)
            .onDisappear {
                dismissedRunIDs.insert(runID)
                presentedRunID = nil
            }
    }
    ```

    FULL SCREEN COVER TIMING: `showLive` is driven by `classifierStateLabel == "SKIING"`. The `onChange` fires on the @MainActor as part of the SwiftUI update cycle. fullScreenCover presents with a system animation. No additional coordination needed.

    LIVE VIEW DISAPPEAR: When the run ends, classifierStateLabel transitions to "CHAIRLIFT" or "IDLE". The onChange fires, sets showLive = false, and the fullScreenCover dismisses. The post-run sheet then presents (triggered by lastFinalizedRunID changing) after the live cover is gone.

    IMPORTANT: UUID is Identifiable (it has an `id: UUID` property returning itself) so `.sheet(item: $presentedRunID)` works correctly when presentedRunID is UUID?.

    Add a #Preview omitting AppModel() constructor if it causes simulator crashes (per Phase 1 precedent).
